# Changelog

本文件记录 QuarkfanTools 应用变更。示例 Skill 的独立历史见
`skills/moje-qa-assistant-basic/CHANGELOG.md` 和
`skills/moje-qa-assistant-adv/CHANGELOG.md`。

## Unreleased

## v2.2.5 - 2026-06-28

- 统一长内容弹窗滚动策略：发布说明、使用手册、帮助说明、预览、会话详情、运行详情、定时任务编辑、Bot 编辑和自定义应用详情等弹窗会固定头部并让内容区滚动，避免配置较多时底部内容被截断。
- 修复 Bot 编辑弹窗中“新增定时任务”点击后看似无反应的问题：新增任务现在先作为前端草稿打开编辑弹窗，填写任务名、Prompt 和投递 `chat_id` 后才保存到配置，取消或关闭会丢弃草稿。
- 运行台、配置、Skill 市场、能力治理、存储管理和定时任务中心改为页内标签组织；新增或编辑配置继续通过弹窗或局部编辑器完成，减少长页面上下翻找。
- 修复 Skill 市场页内标签筛选不生效的问题：标签切换现在会直接按来源或未授权状态重绘列表，并保留搜索框的即时过滤。

## v2.2.4 - 2026-06-28

- 微信未读读取模板改为连续流程：先截取微信列表并让多模态模型识别红点/红色数字徽标，再点击第一个可见未读会话，重新读取打开后的当前会话可见消息。
- 自定义应用运行器支持多模态识别后的受控二段回调，模型 API Key 仍只在主进程配置中使用，不传入自定义应用脚本。
- 微信未读读取模板支持按视觉模型返回的可见未读候选队列连续读取最多 5 个会话；每个打开后的会话会交给多模态模型抽取可见消息，多条消息以当前窗口可见范围为准。
- 存储管理新增自定义应用运行产物统计和清理入口，可清理微信截图等 `apps/<app-id>/` 会话产物；配置页可设置自定义应用产物保留天数和是否自动清理过期产物。
- 自定义应用返回内容新增可配置后处理：默认原样返回，也可让主进程用当前模型配置做一次无工具总结后再回复用户。
- 自定义应用输出协议新增受控投递请求：应用可通过 `deliveries[].routeId` 把处理结果投递到当前 Bot 已启用的投递路由，例如同步到飞书群；脚本拿不到 `chat_id`、凭据或任意发送接口。
- 修复 Bot 编辑弹窗中“新增投递路由”点击后看似无反应的问题：空 `chat_id` 的新路由现在会保留为可编辑草稿，运行时遇到未填写 `chat_id` 的路由只会跳过并写入日志。
- Bot 编辑弹窗的结果投递路由现在直接展示并允许编辑“路由 ID”，用于自定义应用 `deliveries[].routeId`；新增路由默认使用可读的 `delivery-route-N`。
- Bot 编辑弹窗改为左侧分组导航：基础、平台、投递、Skills、能力、命令和定时任务分开配置，切换分组不会丢失未保存输入。
- Bot 编辑弹窗的分组内容区改为独立滚动，长列表分组不会撑出弹窗可视区域。
- 自定义应用回复后处理移到“能力 > 自定义应用”的应用详情弹窗中；每个应用可分别配置是否总结、输入上限和提示词，避免一个应用的总结策略影响其他应用。
- 微信未读读取模板调整点击策略：优先使用 CGEvent 发送全局点击，并在视觉坐标落入窗口范围时优先尝试原始坐标，避免 Retina 截图像素被二次缩放后把第二个会话点成第一个；点击后会重新截图验证目标行和窗口差异，无变化时尝试备用坐标或报告未点中。
- 保持安全边界：点击未读会话可能让微信将该会话标记为已读；模板仍不读取微信数据库、不调用协议、不 Hook 进程、不自动搜索、不自动粘贴、不自动发送。

## v2.2.3 - 2026-06-28

- 为微信未读读取模板新增内置兜底命令 `/wechat-read`。只要 Bot 已授权 `微信未读草稿助手模板`，即使没有手工配置命令映射，也会路由到该自定义应用；未授权时会提示授权能力，而不是误导用户安装 Skill。

## v2.2.2 - 2026-06-28

- 调整微信桌面辅助 PoC 的主链路：内置模板先截取微信当前窗口，通过本地初筛和已配置多模态模型识别可见未读，再可选把回复草稿写入系统剪贴板。
- 保持安全边界：不读取微信数据库、不调用协议、不 Hook 进程、不自动搜索联系人、不自动粘贴、不自动发送。
- 补充未读读取动作计划、受控多模态识别入口和模板 dry-run 测试，便于安装包测试时区分“未授权辅助功能”“本地初筛为空”和“模型未配置”。

## v2.2.1 - 2026-06-28

- 新增微信桌面辅助 Agent PoC：通过自定义应用模板声明 macOS 桌面自动化权限，可尝试激活微信并把回复草稿写入系统剪贴板；当前明确不读取微信数据库、不调用协议、不 Hook 进程，也不自动粘贴或发送。

- 新增面向产品经理接手的 `docs/PRD.md`，集中整理产品定位、版本演进历史、功能全景、权限边界、验收标准、限制和后续路线，并在 README 与 AI 接续入口中加入链接。
- 新增 `docs/PRODUCT_HANDOFF.md`，沉淀产品交接入口、当前边界、1.8.3 客户 Skill 数据目录、企业微信封闭结论、下一阶段产品决策问题和新会话启动指令。
- 补充协作和发版隐性规范：版本号落地前必须保留在 `Unreleased`，形成正式版本号后必须同轮测试和打包，2.x 本地归档只保留最近两个版本，`release/arm64/` 也只保留最近两个版本相关分发文件。

## v2.1.0 - 2026-06-23

- 后续发布链路收敛为 Apple Silicon / arm64：`npm run pack:mac` 默认只生成 arm64 安装包，运行时准备脚本也只准备 arm64 的 lark-cli、wecom-cli 和 Claude runtime；Intel x64 只保留为历史版本兼容样本。
- 旧 `qah` 应用数据迁移前新增本地备份：首次迁移会先复制旧目录到 `quarkfantools/backups/legacy-qah-<timestamp>/`，再迁移配置、workspace 和状态，并用 marker 避免重复迁移。
- 同步 `v1.6.17` 多飞书 Bot 监听修复：飞书 Provider 不再使用单共享事件入口，改为每个 Bot 使用独立 HOME/profile 维护自己的事件订阅，并继续在 Runtime 中按 mention 目标做跨 Bot 路由。此变更修复部分 Intel 客户环境下单共享入口只覆盖后启动 Bot 所属飞书应用的问题。
- 能力页新增扩展治理诊断，集中展示自定义应用入口和权限风险、套件缺失依赖、Workflow 步骤引用缺失，并给出授权前处理建议。
- 套件新增版本、发布者、可信来源和标签展示，并支持同 ID 升级、卸载、manifest 诊断和被 Bot、命令或定时任务引用时的卸载保护。
- 自定义应用新增本机生命周期收口：导入会记录安装状态，支持同 ID 升级、卸载、manifest 阻断诊断和被 Bot/套件引用时的卸载保护。
- 能力执行链新增 Owner 审批阻断：命令命中 `requireOwnerApproval` 时先私聊 Owner，定时任务命中时失败并记录原因。
- 能力页新增 Bot 治理控制台和能力使用审计报表，按 Bot 展示授权引用、policy、命令/定时绑定和最近能力调用结果；命令、定时任务和审批阻断会写入 `capability-audit.jsonl`。
- Bot 编辑弹窗新增能力运行策略配置，可为 MCP、自定义应用和套件分别控制 Agent、命令、定时任务和 Owner 审批边界。
- Workflow 声明式步骤新增输入模板、单步超时和重试次数，步骤日志与定时任务摘要会展示重试尝试次数。
- Workflow 声明式步骤新增步骤输出变量、条件跳过、循环和失败恢复，跳过步骤会进入步骤事件和运行记录摘要。
- 命令映射新增别名、MCP 目标和 `/help` 自动帮助列表，用户可查看当前 Bot 已启用命令、别名和说明；命令配置区会提示保留命令和命令名/别名冲突。
- 定时任务新增 5 段 cron 表达式计划类型，支持 `*`、列表、范围和步进；Bot 编辑弹窗中的定时任务配置改为计划、目标、投递分区展示，并按选择隐藏无关字段。
- Bot 编辑弹窗中的定时任务改为列表 + 编辑弹窗模式；列表直接提供“立即执行”“编辑”“删除”，详细计划、目标和投递参数进入弹窗配置。
- 定时任务新增失败重试治理：计划触发失败可按配置延迟重试，连续失败超过上限后暂停自动排期并展示暂停原因。
- 定时任务新增 MCP capability 目标、失败 chat 告警和独立“定时任务”中心，可集中查看任务状态、下次计划、立即执行和运行历史。
- 定时任务定义与运行态字段分离：配置只保存任务定义，Bot 状态目录只保存运行态；升级时兼容旧版完整任务状态文件，避免旧状态或回滚升级覆盖当前任务定义。
- MCP 协议诊断新增持久化探测日志：刷新诊断时会把握手结果、工具名摘要、退出码、signal 和 stderr 尾部写入 `state/mcp-diagnostics.jsonl`，能力页会展示最近一次探测结果。
- MCP 配置新增 HTTP / SSE 占位传输，可提前保存 URL 并在能力页提示“运行时未接入”；Agent、命令和定时任务仍只会注入或列出已授权的 `stdio` MCP。
- 能力页新增 IM / CONNECTORS 诊断，按 Bot 检查企业微信事件桥、飞书知识连接器、投递路由和钉钉占位状态。
- 应用内使用手册新增场景化教程，覆盖飞书问答、企业微信接入、命令、定时任务、套件/Workflow、MCP 和升级恢复；配置页补充 MCP 传输、命令、URL、环境变量等 `?` 说明。
- 预留能力新增友好建设中反馈：钉钉、HTTP/SSE MCP、webview/mcp-adapter/workflow 自定义应用入口不会被误列为可执行目标，旧配置命中时会给出明确不可执行原因。
- 存储管理的文件缓存索引支持单条删除、90 天自动失效、新鲜度状态和索引校验修复；共享同一内容 hash 的缓存会保留到最后一个索引引用被删除。
- 存储管理会话详情支持事件筛选和 JSON 导出；定时任务运行历史改为摘要列表加详情弹窗，避免长详情撑开页面。

## v2.0.3 - 2026-06-20

- 同步 `v1.6.16` 多飞书 Bot 修复：飞书事件监听改为同一应用进程内共享入口，收到事件后按 mention 目标路由到唯一目标 Bot，避免多个 `lark-cli event +subscribe` 长连接被飞书服务端分流。

- 运行台点击启动会立即写入本地日志；主进程记录启动请求和飞书身份确认阶段，lark-cli 配置校验、初始化和 keychain-downgrade 短命令增加 30 秒超时。
- 2.0 的企业微信 Provider 仍保持独立事件桥；本次共享入口只作用于飞书 Provider，不改变企业微信和后续钉钉的 Provider 边界。

## v2.0.1 - 2026-06-19

- 修正多飞书 Bot 群聊路由策略：现场事件确认 `mentions.id.open_id` 可能不同于 bot info 的 `open_id`，现在有 mention 时按 mention 名称、App ID、应用名和 open_id 等目标值匹配。
- 有 `mentions` 的群聊消息不再用事件头 `sourceAppId` 判定目标 Bot；`sourceAppId` 只在缺少 mention 元数据的旧事件中兜底，避免正确机器人被误忽略。
- 将同一修复同步到 1.6 维护线并发布 `v1.6.10`，避免 2.0 后续重复修复同类问题。

## v2.0.0 - 2026-06-19

- 将 `v1.6.9` 多飞书 Bot 群聊艾特修复带入 2.0：飞书 Bot 启动时确认真实 `open_id` 和应用名，群聊消息先按 mention 目标过滤，避免多个飞书 Bot 同时回复。
- 记录 `aid=552564` 调查结论：该值是飞书事件 WebSocket endpoint 的连接层参数，不是开放平台 `cli_...` App ID，不能用于 Bot 身份判断。
- 新增 Bot 级 `/xxx` 命令映射，可将命令路由到已授权 Skill、已授权套件、套件派生 Workflow 或已授权自定义应用，并支持 `{{args}}` Prompt 模板。
- 新增套件能力：支持导入包含 `suite.json` 的目录、在能力页预览、在 Bot 编辑弹窗中挂载授权，并可作为命令目标注入套件上下文执行。
- 新增 Bot 级定时任务：支持 `interval/daily/weekly` 计划、`agent/command/capability` 目标、本机调度与运行记录，并可把结果投递到指定 `chat_id`。
- 定时任务的 `capability` 目标现已支持已授权套件；Skill/Suite/App 的 capability 执行链已统一，并补齐 `allowScheduledUse` 治理校验。
- Runtime Binding 新增统一 capability executor，为后续 workflow、mcp-adapter 等可执行能力扩展预留稳定入口。
- Runtime Binding 进一步拆分为 executable capability binding resolver 与 capability executor，命令和定时任务不再直接分支 Skill/Suite/App 细节。
- 新增 suite 派生 Workflow 能力：已授权套件下的 Workflow 可直接作为命令和定时任务的 capability 目标执行。
- Workflow 支持声明式步骤编排：当前可按顺序执行 prompt step 和 capability step，并把上一步输出传给下一步。
- Workflow 步骤执行现在进入运行台日志；定时任务触发的 Workflow 会把步骤状态和短输出摘要写入运行记录。
- 存储管理新增定时任务最近运行历史，可查看 Bot、任务、状态、耗时和详情；Workflow 定时任务会显示步骤摘要。
- Bot 定时任务支持手动立即运行已保存且启用的任务，结果进入同一运行历史，且不会扰动原本已计算的下一次计划时间。
- 新增 MCP 能力：支持全局 `stdio` 配置、Bot 维度授权，并以严格 MCP 配置模式注入 Claude Agent SDK。
- 能力页新增 MCP 静态诊断，展示命令解析、cwd、环境变量和 Bot 授权问题，并以 OK/WARN/ERROR 标记配置状态。
- MCP 诊断刷新会短暂启动 `stdio` 服务执行协议握手和 `tools/list`，在能力页预览工具名，并在失败时展示协议错误、退出码和 stderr 尾部。
- 新增多 IM Provider 底座，Bot 可选择飞书或企业微信作为消息平台，并通过飞书知识连接器和投递路由支持“企业微信接收、飞书查资料、结果转发飞书”的两端场景；钉钉 Provider 已预留结构。
- 打包链路新增官方 `@wecom/cli` macOS universal runtime，arm64 与 x64 安装包都会携带 `wecom-cli`。
- 打包链路会从本地 `logo.png` 生成 macOS app icon，安装包不再使用 Electron 默认图标。
- Skill、自定义应用和套件卡片支持打开资源所在目录，便于复制、检查或备份资源文件。
- Bot 支持配置长任务自动提示秒数和文案，单次消息处理超时仍未完成时会先回复一次提示，最终结果仍正常回复。
- 存储管理会话详情新增结构化消息明细时间线，区分接收消息、Agent 可观察工作过程、长任务自动提示和最终回复，并继续展示 workspace 文件清单。
- 补充 2.0.0 原始范围完成度矩阵，明确能力治理、自定义应用、套件、Workflow、命令、定时任务、MCP、缓存和发布的已完成与未完成边界。

- 2.0.0 能力治理底座：新增 Bot `capabilityRefs` 配置、统一 Capability Registry/Resolver，并兼容旧版 `skillNames` 授权。
- 新增自定义应用能力：可导入包含 `app.json` 的本地应用目录，进入能力页预览，并在 Bot 编辑弹窗中显式授权。
- 存储管理拆分会话数据和文件缓存，可单独清理应用级内容哈希缓存。
- 下载前缓存命中开始落地：飞书消息附件下载会先按 Bot 和资源 key 查询文件缓存，命中时直接复用，避免同一 Bot 跨会话重复下载同一附件。
- 新增受控飞书文件缓存协议：Agent 可通过 `LARK_CACHED_FILE` 请求主进程下载云盘文件或导出云文档，优先复用应用级文件缓存，未命中再下载并回灌给同一会话继续分析。
- Runtime 会拦截 Agent 裸调 `lark-cli drive +download` 或 `drive +export` 的 Bash 操作，避免绕过受控文件缓存 helper。
- 存储管理中的定时任务运行历史支持按 Bot 和状态筛选；能力页 MCP 诊断支持手动刷新。
- 存储管理新增文件缓存索引只读视图，可按 Bot 和来源类型筛选消息附件、云盘下载和云文档导出缓存。
- 新增应用级主题切换，支持跟随系统、浅色和深色；左侧品牌区开始使用 `logo.png`。
- 左上角应用内使用手册新增“能力与自定义应用”、套件命令和主题说明。
