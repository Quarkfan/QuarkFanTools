# 2.0 端到端验证清单

本清单只记录需要真实飞书、企业微信、MCP 服务或发布环境配合的验证项。1.x 系列已封版，后续不再作为同步目标。

## 飞书真实消息

- [ ] 单飞书 Bot 私聊消息：收到消息、添加原消息表情、执行 Agent、回复结果、移除表情。
- [ ] 单飞书 Bot 群聊艾特：只响应被艾特消息，未艾特消息不回复。
- [ ] 多飞书 Bot 同时运行：每个 Bot 使用独立 HOME/profile 订阅事件。
- [ ] 多飞书 Bot 群聊交叉投递：按 mention 目标路由到唯一 Bot，其他 Bot 记录忽略诊断。
- [ ] mention open_id 与 bot info open_id 不一致时：按 mention 名称、App ID 或应用名仍能命中正确 Bot。
- [ ] 用户态 OAuth：每个 Bot 独立授权后可搜索、读取、导出飞书资料。
- [ ] 附件消息：图片进入多模态输入，Office 文件完成预处理。

## 企业微信真实环境

- [ ] 企业微信 Provider 当前暂时封闭：配置页显示封闭说明，企业微信消息平台、CLI 缓存初始化、聊天列表获取、事件桥和轮询配置不可操作。
- [ ] 已有旧企业微信 Bot 配置启动时必须给出“企业微信 Provider 暂时封闭”提示，不启动监听、轮询或投递。
- [ ] 能力页 IM / CONNECTORS 诊断对企业微信 Bot 展示封闭状态，并建议改用飞书。
- [ ] 能力页 IM / CONNECTORS 诊断与真实配置一致：事件桥、飞书连接器、投递 `chat_id` 缺失时能给出可操作提示，补齐后状态恢复。

## 命令、Workflow 和定时任务

- [ ] `/xxx` 命令调用已授权 Skill。
- [ ] `/xxx` 命令调用已授权套件。
- [ ] `/xxx` 命令调用套件派生 Workflow。
- [ ] `/xxx` 命令调用声明 `commandCallable=true` 的自定义应用。
- [ ] 定时任务按 `interval/daily/weekly/cron` 触发并投递 chat。
- [ ] 定时任务调用 `agent/command/capability` 三类目标。
- [ ] 定时任务失败后按 retry 配置重试，超过上限后暂停自动排期。
- [ ] 定时任务手动立即执行进入运行历史，且不扰动原自动计划。

## MCP

- [ ] 真实 stdio MCP 服务可通过静态诊断和协议探测。
- [ ] 真实 stdio MCP 工具进入 Claude Agent SDK 严格配置并可被 Agent 调用。
- [ ] MCP 启动失败时，能力页展示错误、退出码、signal 和 stderr 尾部。
- [ ] 应用重启后，能力页仍展示最近一次 MCP 探测摘要。
- [ ] HTTP / SSE MCP 只显示占位诊断，不进入 Agent、命令目标或定时任务目标。
- [ ] 后续真正接入 HTTP / SSE 后，补充远端鉴权、证书、连通性和工具列表验证。

## 存储和清理

- [ ] 会话详情能展示真实消息轮次、Claude session、Agent 工作过程、长任务提示、最终回复和 workspace 文件。
- [ ] 过期会话清理不删除用户配置、飞书授权、用户 Skills 或应用级文件缓存。
- [ ] 文件缓存命中后，大文件不会重复下载。
- [ ] 文件缓存单条删除只删除对应索引；共享 hash 仍被其他索引引用时内容目录保留。

## 发布交付

- [ ] arm64 安装包可在 Apple Silicon macOS 启动并完成基础配置。
- [ ] 从旧版数据目录升级时，确认 `backups/legacy-qah-<timestamp>/` 已生成且配置、workspace、state 迁移后可用。
- [ ] 安装包内置 lark-cli、wecom-cli 和 Claude runtime 架构正确。
- [ ] 签名和公证完成后，macOS 安装提示符合预期。
